# Ticket 001: TeleTalk MVP

**Reference**: [thoughts/research/initial.md](../research/initial.md)

## Goal

Build a working macOS menu bar app where the user holds a hotkey, speaks, releases, and transcribed text appears at their cursor. Fully local, no cloud, no subscription. English only.

## Success Criteria

- [ ] App lives in the menu bar, launches at login optionally
- [ ] User can hold a configurable hotkey to record, release to transcribe (hold-to-talk mode)
- [ ] User can press hotkey once to start, once to stop (toggle mode)
- [ ] A small floating overlay shows recording/transcribing state
- [ ] Transcribed text is inserted at the active cursor position in any app
- [ ] Parakeet TDT 0.6B v2 model downloads on first launch (~600MB)
- [ ] Everything runs locally on Apple Neural Engine — no network calls after model download
- [ ] macOS 14.0+ (Sonoma), Apple Silicon only

## Non-Goals (MVP)

- AI text cleanup / filler word removal (future feature)
- Multilingual support
- Streaming/partial transcription preview
- Context-aware tone adaptation
- Personal dictionary / snippets
- App Store distribution
- Intel Mac support

## Implementation Plan

### Phase 1: Project Scaffolding

**1.1 — Xcode project setup**
- Create a new macOS app target (Swift, SwiftUI lifecycle)
- Configure as menu bar app (`NSStatusItem`) — no dock icon
- Disable App Sandbox in entitlements (required for accessibility + input monitoring)
- Set deployment target to macOS 14.0
- Add `NSMicrophoneUsageDescription` to Info.plist
- Set up signing with Developer ID (not App Store)

**1.2 — Swift Package dependencies**
Add via SPM:
- `FluidAudio` — Parakeet TDT transcription on ANE ([github.com/FluidInference/FluidAudio](https://github.com/FluidInference/FluidAudio))
- `KeyboardShortcuts` — global hotkey registration and UI ([github.com/sindresorhus/KeyboardShortcuts](https://github.com/sindresorhus/KeyboardShortcuts))
- `LaunchAtLogin` — login item support ([github.com/sindresorhus/LaunchAtLogin](https://github.com/sindresorhus/LaunchAtLogin))

Verify: project builds, menu bar icon appears, app runs.

---

### Phase 2: Model Management

**2.1 — Model downloader**
- On first launch, check if Parakeet TDT 0.6B v2 CoreML model exists in `~/Library/Application Support/TeleTalk/Models/`
- If missing, show a one-time setup screen: "TeleTalk needs to download the speech model (~600MB). This is a one-time download."
- Download from HuggingFace: [FluidInference/parakeet-tdt-0.6b-v2-coreml](https://huggingface.co/FluidInference/parakeet-tdt-0.6b-v2-coreml)
- Show progress bar during download
- Verify integrity after download (checksum)
- Store model path in UserDefaults

**2.2 — Model loading**
- Load model via FluidAudio SDK on app launch (after download complete)
- Keep model loaded in memory for instant transcription
- Handle errors gracefully: corrupt model → re-download prompt

Verify: model downloads, loads, app shows "Ready" state.

---

### Phase 3: Audio Capture

**3.1 — Microphone permission flow**
- On first use, request microphone access via `AVCaptureDevice.requestAccess(for: .audio)`
- If denied, show alert directing user to System Settings → Privacy → Microphone
- Gate recording behind permission check

**3.2 — Audio recording engine**
- Use `AVAudioEngine` with input node
- Configure format: 16kHz mono PCM (Parakeet's expected input format)
- On recording start: install tap on input node, accumulate samples into a buffer
- On recording stop: remove tap, finalize buffer
- Handle edge cases: no mic connected, mic disconnected mid-recording

Verify: can record audio, play it back, confirm waveform looks correct.

---

### Phase 4: Global Hotkey

**4.1 — Hotkey registration**
- Use `KeyboardShortcuts` package for registration and settings UI
- Default hotkey: `Ctrl+Shift+Space` (avoid Fn — it's unreliable to intercept)
- Two named shortcuts:
  - `holdToTalk` — hold to record, release to transcribe
  - `toggleRecord` — press to start, press again to stop
- Store user's preference for which mode is active

**4.2 — Hold-to-talk implementation**
- This is trickier — `KeyboardShortcuts` fires on keyDown/keyUp
- On keyDown: start recording + show overlay
- On keyUp: stop recording + trigger transcription
- Debounce: ignore keyDown events shorter than ~200ms (accidental taps)

**4.3 — Toggle mode implementation**
- First press: start recording + show overlay
- Second press: stop recording + trigger transcription
- Visual distinction: overlay shows elapsed time in toggle mode

**4.4 — Accessibility permission check**
- Check `AXIsProcessTrusted()` on launch
- If not trusted, prompt user to grant Accessibility access in System Settings
- Provide a button that opens `System Settings → Privacy → Accessibility` directly

Verify: hotkey triggers recording in both modes from any app. Works while other apps are focused.

---

### Phase 5: Transcription

**5.1 — FluidAudio integration**
- Initialize `FluidAudio.ASRManager` with the downloaded Parakeet TDT v2 model
- On recording stop, pass the PCM audio buffer to the ASR manager
- Receive transcribed text string
- Measure and log latency (target: <500ms for a 10-second utterance)

**5.2 — Error handling**
- Empty audio (user pressed and released immediately): no-op, dismiss overlay silently
- Model not loaded: show error in overlay, log
- Transcription returns empty: show brief "Nothing heard" in overlay

Verify: speak a sentence, get text back. Check accuracy against known phrases.

---

### Phase 6: Text Insertion

**6.1 — Accessibility API insertion (primary)**
- `AXUIElementCreateSystemWide()` → query `kAXFocusedUIElementAttribute`
- Check if focused element supports `kAXValueAttribute` or `kAXSelectedTextRangeAttribute`
- If it has a selection range, insert at cursor position
- If it supports value, append/set value appropriately

**6.2 — Clipboard paste fallback**
- If AXUIElement approach fails (some apps don't expose accessibility properly):
  - Save current clipboard contents
  - Write transcribed text to `NSPasteboard.general`
  - Simulate `Cmd+V` via `CGEvent` (keyDown + keyUp, `.maskCommand` flag)
  - Restore original clipboard contents after a short delay (~200ms)

**6.3 — Insertion strategy selection**
- Try accessibility API first
- If it throws or the focused element isn't a text field, fall back to clipboard paste
- Log which method was used for debugging

Verify: transcribed text appears in TextEdit, VS Code, Slack, Safari text fields, Terminal.

---

### Phase 7: Overlay UI

**7.1 — Floating overlay window**
- `NSPanel` with `.floating` level, `styleMask: [.borderless]`
- Non-activating (`.nonactivatingPanel`) — must not steal focus from the current app
- Ignore mouse events except for a cancel/close button
- Position: bottom-center of the screen, ~100px from bottom edge
- Size: small pill shape (~200x40px)
- Transparent background with vibrancy/blur (`.hudWindow` or custom `NSVisualEffectView`)

**7.2 — States and transitions**
- **Idle**: overlay hidden
- **Listening**: overlay appears with mic icon + "Listening..." label + subtle pulse animation
- **Transcribing**: overlay changes to spinner + "Transcribing..." label
- **Done**: overlay fades out after text insertion (~300ms)
- **Error**: overlay shows brief error message, fades out after 2 seconds

**7.3 — SwiftUI overlay content**
- Build the pill UI in SwiftUI, host in the NSPanel via `NSHostingView`
- Animate state transitions

Verify: overlay appears/disappears correctly, doesn't steal focus, looks native.

---

### Phase 8: Menu Bar & Settings

**8.1 — Menu bar item**
- `NSStatusItem` with a mic icon (SF Symbol: `mic.fill`)
- Click opens a menu with:
  - Status line: "Ready" / "Model downloading..." / "No mic access"
  - "Settings..." → opens settings window
  - "Quit TeleTalk"

**8.2 — Settings window**
- SwiftUI window with tabs/sections:
  - **Hotkey**: configure shortcut + choose mode (hold-to-talk / toggle)
  - **General**: launch at login toggle, audio input device picker
- Keep it minimal — no settings bloat

Verify: settings persist across restarts, hotkey changes take effect immediately.

---

### Phase 9: Polish & Edge Cases

**9.1 — First-run experience**
- On first launch:
  1. Request microphone permission
  2. Check/request accessibility permission
  3. Trigger model download
  4. Show brief "You're ready! Use [hotkey] to start dictating." message
- Each step has clear error recovery if the user denies/cancels

**9.2 — Permission recovery**
- If permissions are revoked after initial grant, detect and show a non-intrusive alert
- Menu bar status updates to reflect missing permissions

**9.3 — Audio edge cases**
- Handle mic disconnection mid-recording: stop recording, show error
- Handle very long recordings (>2 minutes): warn or auto-stop with a configurable limit
- Handle system audio interruptions (e.g., incoming call)

**9.4 — Memory management**
- Model stays loaded while app is running (fast subsequent transcriptions)
- Monitor memory usage — Parakeet v2 CoreML model should be manageable on 8GB machines
- If memory pressure is high, consider lazy loading (load on first transcription, not launch)

## Risks

| Risk | Mitigation |
|------|------------|
| FluidAudio API instability (relatively new SDK) | Pin version, read VoiceInk's integration as reference |
| Hold-to-talk keyUp detection unreliable with `KeyboardShortcuts` | Fall back to `CGEvent` tap directly if needed |
| AXUIElement text insertion fails in some apps (Electron, etc.) | Clipboard paste fallback covers most cases |
| 600MB model download fails / slow connection | Resume support, checksum verification, retry logic |
| 8GB RAM machines struggle with model in memory | Lazy load, measure actual memory footprint early |
| Parakeet v2 CoreML model not available or changes URL | Cache the HuggingFace URL, version-pin the model |

## Testing Plan

1. **Unit**: AudioRecorder buffer format, TextInserter logic, ModelManager path handling
2. **Integration**: Full flow — hotkey → record → transcribe → insert — in at least 5 apps (TextEdit, VS Code, Slack, Safari, Terminal)
3. **Edge cases**: No mic, permission denied, model missing, empty recording, very long recording, rapid hotkey toggling
4. **Performance**: Measure transcription latency across recording lengths (1s, 5s, 15s, 30s, 60s)
5. **Memory**: Profile on 8GB and 16GB machines

---

## Definition of Done

The MVP is done when a user can:
1. Install the app (drag to /Applications)
2. Grant mic + accessibility permissions via guided setup
3. Wait for model to download once
4. Press and hold `Ctrl+Shift+Space`, speak a sentence, release
5. See their words appear at the cursor in any text field
6. The entire flow works offline with no cloud dependency
